## Code Generation — Do Not Edit Generated Files Directly

### README.md (helm-docs)

README.md files for `document-engine` and `ai-assistant` are **auto-generated** by
[helm-docs](https://github.com/norwoodj/helm-docs). CI fails if they are out of sync.

- **Template**: `charts/<chart>/README.md.gotmpl`
- **Source**: `Chart.yaml` metadata + `values.yaml` comments
- **Do not edit `README.md` directly** — edit `README.md.gotmpl` or `values.yaml` instead

To regenerate locally:

```bash
helm-docs --chart-search-root charts/document-engine
helm-docs --chart-search-root charts/ai-assistant
```

### values.schema.json (helm-values-schema-json)

JSON Schema files are generated by
[helm-values-schema-json](https://github.com/losisin/helm-values-schema-json).

- **Config**: `charts/<chart>/.schema.yaml`
- **Source**: `values.yaml` with schema annotations

To regenerate locally:

```bash
cd charts/document-engine && helm schema -input values.yaml -draft 2020 -indent 2 -output values.schema.json
cd charts/ai-assistant && helm schema -input values.yaml -draft 2020 -indent 2 -output values.schema.json
```

## values.yaml Comment Conventions

These annotations drive README and schema generation:

| Annotation | Purpose | Example |
|------------|---------|---------|
| `# --` | Parameter description (appears in README) | `# -- Enable JWT` |
| `# @section -- XX. Name` | Groups parameters into README sections | `# @section -- 01. API authentication` |
| `# @schema enum: [A, B]` | Adds enum constraint to JSON schema | `# @schema enum: [RS256, RS512]` |
| `# @notationType -- reference` | Controls how default is rendered in README | `# @notationType -- reference` |
| `# @default -- none` | Overrides displayed default in README | `# @default -- none` |
| `# @ignored` | Excludes parameter from README table | `# @ignored` |
| `# (object)` | Type hint for helm-docs | `# -- (object) JWT settings` |

## CI Workflows

Three GitHub Actions workflows run on this repository:

1. **Documentation** (`helm-docs.yaml`) — PR trigger. Validates README.md is in sync with values.yaml. Fails on diff.
2. **Lint and Test** (`lint-test.yaml`) — PR trigger. Runs `ct lint` and `ct install` against a Kind cluster with CloudNativePG and MinIO pre-installed.
3. **Release** (`release.yml`) — Push to `master`. Publishes chart releases via chart-releaser and updates GitHub release notes from CHANGELOG.md.

## Local Development

### Linting

```bash
ct lint --target-branch master
```

Or for a single chart:

```bash
helm lint charts/document-engine
helm lint charts/document-engine -f charts/document-engine/ci/02-cnpg-clustering-values.yaml
```

### Template Rendering

```bash
helm template test-release charts/document-engine
helm template test-release charts/document-engine --set workloadType=StatefulSet
```

### Dependency Management

Charts with dependencies (e.g., `ai-assistant` depends on `document-engine`):

```bash
helm dependency update charts/ai-assistant
```

Dependency chart archives in `charts/*/charts/` are gitignored.

## Versioning

- Chart version in `Chart.yaml` follows semver
- **Major bump**: breaking changes (new workload types, removed values, schema changes)
- **Minor bump**: new features, backward-compatible additions
- **Patch bump**: bug fixes, documentation
- Update `CHANGELOG.md` for every version bump (used in GitHub release notes)

## Pre-commit Checklist

After modifying any chart, regenerate derived files before committing:

1. **README.md** — run `helm-docs` whenever `values.yaml`, `Chart.yaml`, or `README.md.gotmpl` change:
   ```bash
   helm-docs --chart-search-root charts/document-engine
   helm-docs --chart-search-root charts/ai-assistant
   ```
2. **values.schema.json** — run `helm schema` whenever `values.yaml` schema annotations change:
   ```bash
   cd charts/document-engine && helm schema -input values.yaml -draft 2020 -indent 2 -output values.schema.json
   ```

CI will fail if generated files are out of sync with their sources.

## Template Patterns

### Shared Helpers (`_helpers.tpl`)

Standard helpers for names, labels, selectors plus:
- `document-engine.isStatefulSet` — returns truthy when `workloadType` is `StatefulSet`
- `document-engine.scheduling` — shared pod scheduling block (nodeSelector, affinity, tolerations, etc.)
- `document-engine.certificateTrust.customCertificates.volumes` / `.volumeMounts` — shared cert trust volumes

### Pod Template Extraction

`document-engine` uses `_pod-template.tpl` to share the pod spec between `deployment.yaml` and `statefulset.yaml`. Both workload types include it via `{{ include "document-engine.podTemplate" . }}`.

### Schema Defaults

All default values come from `values.schema.json`. Templates reference `.Values.*` directly without `| default` fallbacks.
